Java VM Requirements

Last updated: [98/0527 Bob]

Introduction

The characteristics of the Java Virtual Machine we will ship with can (and probably will) have a significant impact on the design of higher level parts of our system. This document is intended as a central clearing point for those features which are promised in future versions of the JVM that we might be able to use.

As a goal, we'd like to be on top of the VM current at the point that we 'freeze' our code (i.e. at the end of RobJ's first stack of infrastructure milestones).

Requirements

1. java.lang.ref

The current resource management pre-design could make good use of the various "weak" references so that reference counting etc. can be avoided. This won't be available until 1.2 (all the current IDEs are only/barely up to 1.1.5, let alone 1.1.6).
