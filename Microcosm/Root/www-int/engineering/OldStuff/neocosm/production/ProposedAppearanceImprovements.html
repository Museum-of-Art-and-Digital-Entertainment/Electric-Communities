Proposed Appearance Improvements

Last updated: [19 May 1998, Tony]

Author(s): Tony, Wolf, Randy.

Introduction

This document is meant to capture things we'd like to improve about our Appearance model.

This will need some fleshing out, but I at least wanted to generate a list of things we've already talked about.

Gesture chaining

It would be really nice if all gestures had a field like playWhenDone in which we could specify another gesture that would immediately follow.

Eventable gestures

Pick up a drink. Select sip from the drink's menu. It would be great if that could trigger a gesture on the avatar that puts the drink up to the mouth. Another example would be sitting. Click 'Sit' on a chair and the avatar walks there, turns appropriately, and then sits.

Play and hold

Often times, we'd like to play a gesture and have it remain in its ending state. By default, our current model will always play a default gesture (Stand, for example) after any other gesture completes.

Path planning

This is obviously more a dynamics engine thing than an appearance thing, although objects will likely need to specify a bounding box or at least a footprint in order for path planning to work.

Redundancy of sound support

We currently support VRML animation and texture animation for 3D objects. 2D objects do cel animation. All types contain syntax for sound support. It would be nice to abstract sound info into a form that could be referenced by all the other gesture types. One way to do this would be to make sounds their own gesture type and then include on all gestures a field called 'playAlso' which would simultaneously trigger another gesture.

Better VRML support
