Introduction

Account Manger is the user account manager window. It is through the Account Manger that user starts, opens, and removes avatars from user's identity.

Related Documents

Cosm startup.

Ingredient and unum factories.

Requirements

Requirements currently implemented:

  1. GUI presenter for the login window.
  2. ELogin.
  3. PassphraseVerifier.
  4. MCAvatarDB
  5. Encripted avatar props
  6. ECApplication
  7. Netscape.application

Architecture

 Current Architecture Overview

Account Manager's architecture is quite simple, consisting of only one class and a handful of methods to manipulate the account manager window.

Proposed Architecture Overview

Same as above. This architecture is working fine currently.

 

Off the shelf alternatives

None.

Other Design Objectives, Constraints and Assumptions

  1. Allow user to change avatar name without affecting any other avatar information or status.
  2. Allow multiple selection.

Current implementation

AccountManager is the only class and contains, in addition to its constructor, methods to handle events, control the tab key behavior, manipulate the window and verify avatar encription.

Which directories on our tree does this subsystem cover?

The source for cosm1/ui/gui/login/crew/AccountManger.java

Examples

AccountManager is automatically invoked when starting ECHabitats. It allows user to create a new avatar, to open an existing avatar or to remove avatars. It behaves like any standard windows window.

 

Testing and Debugging

Being such a simple and small subsystem, its test has been straightforward and there are no known bugs..
