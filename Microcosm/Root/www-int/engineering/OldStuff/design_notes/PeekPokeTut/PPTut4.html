The PeekPoke Tutorial

 

Step 1. Determine Una components

Start the code for Pluribus una with an una structure that contains two unum kinds as depicted in Figure 2 and call them, to say, uPeek and uPoke. To determine the unum kind it is necessary to identify the messages needed to communicate with the unum. The unum kind is as follows.

	// ----- Create Unum structure kind -----

	// unum kind for peekable
	export kind ukPeekable {
	  uPeek(String whoPeeked);
	}

	// unum kind for pokeable
	export kind ukPokeable {
	  uPoke(String whoPoked);
	}

The kind structure is defined, and behaves like a java interface. Note that an unum kind implements a single ingredient and the definition of such ingredients takes place in the ingredient implementation. The export keyword indicates that the associated unum kind is accessible beyond the scope determined by a pluribus unit; where a unit is equivalent to a java package and is discussed later in this tutorial. This makes possible for the unum to actually communicate externally.

The Unum Structure Kind groups the unum kinds that unum structures will implement. The code is as follows.

	// unum structure kind for PeekPokeAble
	export kind PeekPokeAble {
	  kind ukPeekable;
	  kind ukPokeable;
	}
