Step 10. Runing PeekPokeShip

Two more files are needed to run PeekPokeShip on a Solaris environmet: GNUmakefile compiles the code and RunMe executes it. The GNUmakefile is as listed below. PACKAGE sholud be customized.


	PACKAGE		= ec.pl.examples.PeekPoke
	MODNAME		= PeekPokeShip
	FILES.plu = \
	  PeekPokeKinds.plu \
	  PeekablePokeable.plu\
	  PeekerPoker.plu\
	  Main.plu\
	FILES.javac = \
	include $(TOP)/E/makefiles/Defs.gmk
	all: basics
	clean: basic_clean
	include $(TOP)/E/makefiles/Rules.gmk

The RunMe file is listed next. PID1 refers to the host and PID2 refers to the client.


	#!/bin/sh
	# -v
	# java -debug ec.e.start.EBoot ec.pl.runtime.Agency 
		Agent=ec.pl.examples.PeekPoke
	LSERVER=gracie.communities.com:666
	PID1=test1.$$
	PID2=test2.$$
	PID3=test3.$$
	ZZZ="RegisterWith=$LSERVER SearchPath=$LSERVER"
	set -x
	java ec.e.start.EBoot ec.pl.runtime.Agency 
		Agent=ec.pl.examples.PeekPoke PublisherId=$PID1 $ZZZ > output &
	sleep 5
	java ec.e.start.EBoot ec.pl.runtime.Agency 
	    Agent=ec.pl.examples.PeekPoke 
		PublisherId=$PID2 $ZZZ url=e://$LSERVER/$PID1/Doohicky

To compile in Solaris, once the directory is set to your code, simply type make. In the directory, .unit, .unitd and .class files will be generated. To execute, simply type RunMe. The results for the host and clients will be stored in the corresponding files, as specified in RunMe.
