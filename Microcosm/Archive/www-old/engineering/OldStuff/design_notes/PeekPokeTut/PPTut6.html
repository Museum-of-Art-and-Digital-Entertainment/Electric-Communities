Step 3. Determine Ingredients’ Components

Ingredients hold the actual code that implement a property’s behavior, that is, ingredients are an unum's part where the internal structure, appearance, and behavior of presences are determined. As can be seen in Figure 2 each presence structure holds two ingredient structure kinds (isk), one for peekable and one for pokeable, that group the unum and presence kinds. These isk contain the unum and presence kinds previously coded. See, for example, the code for the ingredient structure kind iskPeekable below. It has three components: one that corresponds to the unum for peekable, i.e., ukPeekable, and two for the host and client presences for peekable, i.e., pkHostPeekable and pkClientPeekable respectively.


	// ----- Create Ingredients kind -----
	export kind iskPeekable  {
	  kind ukPeekable;
	  kind pkHostPeekable;
	  kind pkClientPeekable;
	}
	export kind iskPokeable  {
	  kind ukPokeable;
	  kind pkHostPokeable;
	  kind pkClientPokeable;
	}
