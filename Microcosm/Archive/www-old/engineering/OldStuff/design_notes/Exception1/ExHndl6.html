Burlesque: Recursive Catching

Another alternative is to generate some sort of recursive catching in which, when an exception is thrown, a cascaded throw that recurse back until a successful line of computation is reached. The complexity of this operation makes it hard to test and correct behavior prediction becomes quite a challenge, thus a potential source of deep purple.

To revert and reach either an active or a successful line of computation, the value of oldID is set to 0 and used as a flag to indicate that its is one that failed. Thus, when an exception is thrown, the corresponding ecatch either behaves as explained in the 'Andante' section of this document or throws a new exception to continue revertion. 

	eclass Foo {
	  int myID;
	  method uFoo(Bar bar, Bud bud) {
	    myID++;
	    int oldID = myID;
	    etry {
	      bar <- uBar(bud);
	    } ecatch {
	      if(oldID == myID) {
	      	oldID = 0;
	        exceptionRecoveryCode;
	      } else {
	        if (oldID == 0) {
	          ethrow(new RtEException("reverting computation..."));
	        }
	      }
	    }
	  }
	}
