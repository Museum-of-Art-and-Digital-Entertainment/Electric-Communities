Evaluation of JavaSafe

Last Updated:  5/27/98

Author:  Scott Lewis

Introduction

JavaSafe is a source code control system being offerred from JavaSoft.  Here is the home page.

Cost structure for JavaSafe is provided here.  It is not free.
FAQ is here.  Some decent information is here.

According to the FAQ, CVS is based upon RCS, and JavaSafe looks significantly like CVS (for example, it uses the same Copy, Modify, Merge strategy as CVS.

They have emacs additions to allow integration with JavaSafe.   Looks like a nice feature for you emacs fans out there.

They have a feature request mechanism on the web which looks nice.  Don't know how responsive they will be, however.

Some screen shots.  GUI looks OK IMHO, but perhaps a little immature.

Recommendation

Looks like JavaSafe is potentially a useful product.  Given that we are working reasonably effectively with CVS at this point, that JavaSafe is an unknown quantity, and that JavaSafe costs real money (~6K for 25 site license) my recommendation is that we not switch to JavaSafe at this time.  It's technical advantages over CVS at this point (real GUI, some potential for cool features using java server, etc.) are mostly unrealized...and doesn't provide a huge technical advantage over CVS for us right now.   Also it is a very early product, and doesn't have much usage experience behind it yet.  It is therefore likely to be relatively unreliable.

It would be a good idea to track its progress, however, and if it becomes a clear winner over CVS at some point, then we could consider switching to it.  I will track its progress over time.
