DObjectFacet exchange process

  • Requestor:

    • View.requestDObjectCapability(otherID, String requestName, Object data)
    • View creates ‘CapabilityReceipt’ object
    • View sends requestCapability message to otherID, and provides the capability receipt as param

  • Grantor:

    • otherID decides about response (defined by subclass)
      • If yes, stamps receipt with DObjectFacet instance that gives appropriate semantics
    • View.giveDObjectCapability(CapabilityReceipt receipt)
    • View makes sure this is valid capability receipt

  • Requestor:

    • View sends giveCapability message to requestorID
    • DObjectFacet instance provided in giveCapability message
