Measurement Requirements for Neocosm

Created 7/1/98 by Bill

The following is a list of performance measurements we would like to be able to make in Neocosm. It is based on our experiences with R167.

  • Garbage collection rate. Garbage collects can be counted by a garbage object with a finalizer method which adds one to a counter and then creates another instance of the garbage object.
  • Paging rates
  • E Run queue length
  • E Run queue latency - (How long between queuing a message and it's dispatch.)
  • E message processing time - (How long it takes to process a message.)
  • N longest executing messages, with some detail about what objects, methods, and parameters are involved.
  • Synchronization delay for Comm thread to vat communication.
  • Bytes send/received over each comm connection, and total rate for the vat
  • Connection creation/shutdown rate.

We also need a coherent way of recording these measurements for analysis.
