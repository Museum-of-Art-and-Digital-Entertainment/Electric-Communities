Using BUILD1

BUILD1 is a computer dedicated to help you as Build Buddy. The underlying idea is to use it whenever none of our engineers is available to assist you as build buddy, to use it as a third build buddy, or simply to try things on a machine other than your own. It is currently located at the cube next to Alison's, by the windows. Masa maintains this page, please direct your questions and comments to him.

Marking BUILD1 as Busy or Idle
    It is important to let other engineers know when you are utilizing BUILD1 as well as when it is available for them to use. Bitmaps 'busy' and 'idle' on the desktop can help you indicate the status of BUILD1. You can open them with MSPaint by double clicking their corresponding icon. My recommendation is to expand the window to full screen and, using the View menu option, disable Tool Box, Color Box, and Status Bar.

 

Updating src and Media
    To update BUILD1 and ensure it contains the latest src and media directories do the following.

  • Close the src and media directories. This step should not be needed if the previous user did clean up before making BUILD1 idle, but doesn't hurt to double check. Then do any of the following if needed.
  • Select the 4DOS (Build) icon to start up a build shell (Note: as of 12/29/97 you may get 2 messages of the form: 4DOS: eC:\EC\ECENVIRONMENT.BAT [119] Command line too long; these seem to be harmless).
  • Update the build environment directories:
    cd \ cvs update src media
    Alternatively, you can type:
    cvs co src media
    just keep in mind this might take longer than cvs update.
 

tar Up Changes
    To share your changes, independently of the cvs versions on the machines you are using try savechanges.sh and applychanges.sh.

    savechanges.sh packages up all the modified files in a cvs tree into a single tar file. applychanges.sh takes a tar file created by savechanges.sh and updates a cvs tree with the modified files in the tar file. These scripts can be used to share changes between users who each have different versions of a cvs tree.

 

Build and Test
  • If you only need to build the src/cosm1 tree (not E etc.) then build HabiFoo. It assumes you have already checked out the EC and src directories (and the media directory if you build your own repository)
  • If you need to build the entire source tree (include E, Pluribus etc.) then build the entire EC world. It assumes you have already checked out the EC, media and src

  • There are two ways to run HabiFoo to test your changes. You can try Running HabiFoo in stand alone mode for testing changes that don't depend on host/client communication. For tests requiring host/client mode the first thing to do is run habifoo in stand alone mode on the machines involved as explained in Running HabiFoo and then do:
    • Identify either your own computer or BUILD1 as host
    • Teleport the host avatar to the turf where tests will take place
    • Copy the corresponding turf's .mcm file to a location that can be accessed by client.
    • From client double-click host's .mcm file. This will teleport client to host's turf and both host and client will be able to see and interact with each other.

  • If you have to leave BUILD1 alone while your test is still on progress use the 'Busy' bitmap image to let others know BUILD1 is *still* in use.
 

Clean Up When Done
    Once you are done with your testing and before you can allow others to use it make sure to do the following.
  • Remove all the files you generated that don't belong to current standard build.
  • Close all directories.
  • Empty the recycle bin.
  • Go to the Start menu, click Shutdown and select 'Close all programs and log on as a different user'. This ensures the next user won't login as you.
