Here is an example of the consequences of ethrow being a message send.

emethod receive(int value, EDistributor done)
{
    // sanity checking
    if (value < 0) ethrow(new RtEException("bad value"));
     
    // do main body of routine...
    System.out.println("value is " + value);
    done <- forward(etrue);
}

Suppose this message is invoked as:

Obj <- receive(-1, &finished);

The exception will be thrown, but the code will also print "value is -1" and forward the distributor.
