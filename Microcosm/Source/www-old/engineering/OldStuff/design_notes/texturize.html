The Texture Ingredients

Introduction

This design note explains the texture ingredients used to change the texture of una in Microcosm. It describes the design as it was at the time of the PCForum demo, in late March '97.

Overview

There are two ingredients, Texturizer and Texturizable, each of which has an associated ec.cosm.gui.scene.Texture object. Texturizables can have their Textures set and Texturizers can be told to set Texturizables' Textures to the Texturizers'.

At this point in time (4/8/97), that is all there is to it: Texturizables don't actually do anything with their Textures as there was no mechanism a few days before PC Forum to actually communicate to the renderer about texture changes in presented objects. Because of this, the Texture ingredients were not completed; i.e. there is no mechanism for client/server relationships for Texture ingredients.

Texturizable Ingredient

The Texturizer ingredient is of kind ikTexturizable and contains the following information:
  • Texture
The Texturizable's Texture is an ec.cosm.gui.scene.Texture. This Texture is initialized by the Texturizable's init() method. The Texturizable has two methods:

init(Texture textureMap);
iTexturize(Texture textureMap);

init sets the Texturizable's initial Texture to textureMap.
iTexturize changes the Texturizable's Texture to the passed in textureMap.

Texturizer Ingredient

The Texturizer ingredient is of kind ikTexturizer and contains the following information:
  • Texture
The Texturizable's Texture is an ec.cosm.gui.scene.Texture. This Texture is initialized by the Texturizable's init() method. The Texturizable has two methods:

init(Texture textureMap);
iTexturize(kind ikTexturizable target);

init sets the Texturizable's initial Texture to textureMap.
iTexturize sends an iTexturize() message to target with the Texturizer's Texture.

Texturize Problems

  • The final hookup between the Texturizable and the UI ingredient to allow the actual presentation to be changed needs to be added.
  • Any necessary client/server coding needs to be done.
  • Currently Textures are very simple and based upon the java.awt.Colors. These are read out of realm.txt files and ec.cosm.gui.scene.Textures are generated and passed to the init() methods. We need to implement more complex ways of doing textures, with color arrays and bit masks.
  • It all needs to be tested.
