Step 6. Create Presence Structure

In the presence structure all the constituent ingredients of the presence are listed. Two presence ingredient names are declared within a presence structure, one for Peekable and one for Pokeable. Each one of them contains deliver statements to route unum and presence messages to the actual ingredient that implements them. Two presence structures are needed here, one for the host and one for the client (as shown in Figure 2.) 


	// --------------- Create presence structure ---------------
	// psHostPeekablePokeable
	export presence structure psHostPeekablePokeable {
	  kind pskHostPeekablePokeable;
	  ingredient pinHostPeekable {
	    kind iskPeekable;
	    deliver unum ukPeekable;
	    deliver presence pkHostPeekable;
	  }
	  ingredient pinHostPokeable {
	    kind iskPokeable;
	    deliver unum ukPokeable;
	    deliver presence pkHostPokeable;
	  }
	}
	// psClientPeekablePokeable
	export presence structure psClientPeekablePokeable {
	  kind pskClientPeekablePokeable;
	  ingredient pinClientPeekable {
	    kind iskPeekable;
	    deliver unum ukPeekable;
	    deliver presence pkClientPeekable;
	  }
	  ingredient pinClientPokeable {
	    kind iskPokeable;
	    deliver unum ukPokeable;
	    deliver presence pkClientPokeable;
	  }
	}
