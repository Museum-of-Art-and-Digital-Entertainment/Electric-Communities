Molto Moderato: Last-Largest Successful

Another synchronous approach to exception handling.

A variation to the last successful approach consists on keeping track of the last successufl computation that entered the runtime instead of simply the last one to succeed. To do so track over of the last-sent successful line of computation is kept through myID, as shown in the code below. 

	eclass Foo {
	  int myID;
	  Bud myLastBud;
	  Bud myPreviousState;
	  Bud myLastRealityID;
	  method uFoo(Bar bar, Bud bud) {
	    Bud oldBud;
	    myID++;
	    int oldID = myID;
	    oldBud = myLastBud;
	    ewhen successful (...) {
	      myPreviousState = oldBud;
	      myLastRealityID = oldID;
	    }
	    etry {
	      bar <- uBar(bud, &successful);
	    } ecatch {
	      if (oldID > myLastRealityID) {
	        myLastBud = myPreviousState;
	      }
	    }
	  }
	}

As with the 'last-successful line of computation' approach, an ewhen structure to for update when optimistic computation succeeds is needed, thus an important drawback.
