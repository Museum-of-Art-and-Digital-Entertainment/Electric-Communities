DOM Software as it exists

  • Only one conceptual addition to original Unum/Presence model: Session

  • ‘View’ class (abstract) == Local representation of session

    • ‘Region presence’ for all regions
    • Local representation of multipoint session
    • Two ‘roles’ possible for Views: group manager and other
    • Maintains group membership in session
    • Manages presence creation/destruction
    • Provides basic notification service (object arrival/departure)
    • Provides reliable multicast messaging to remote presences
