DObjectFacet Exchange Result

  • Dobjects have facet(s) of arbitrary semantics to other Dobjects within context of same View

  • View can revoke (e.g. on partition)

  • Grantor can revoke as necessary

  • Already built ‘auto-exchange

    • Using CRAPI, all ‘public’ methods automatically exposed as facet to all other Dobjects
    • Subclasses define a method as public, and it will then be exposed to other Dobjects for asynchronous invokation

  • We could use other criteria for exposure (e.g. all E object methods?)
