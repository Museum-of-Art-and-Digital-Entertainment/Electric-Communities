    The real world's understanding of identity. An (the?) owner. The abstract object that represents the user's session. May be associated with a body (i.e., Avatar or Observer). Public_access_container

Description

    Holds objects. User can open/close it. It can be locked open or locked closed... which we are now calling "stuck open" or "stuck closed". The contents can be locked: when the container is open it can be configured such that the objects cannot be removed. Get and Put can be "locked" independently: e.g. The container can be set such that objects can be Put In the container, but not removed. Likewise, the container may allow objects to be removed, but objects cannot be placed in the container. It may display its contents in a viewer window, or in the main window. You can look at it, carry it, give it, glue it, move it, name it, own it, can make a cyclical animation and a cyclical ambient sound (set at instantiation time).

Stateful Prop

Description

    This is a prop that maintains some state. User can change the state though a menu action, which results in the prop displaying a different image representing the new state. There can be N states. To possible controls for the next state are: - increment - decrement - choose specific state. Additionally: optional animation/sound sequence between state changes, where the animation can be specific to the current state and new state.
