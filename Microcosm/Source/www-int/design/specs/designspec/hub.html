Randy's (Yet another) hub tools taxonomy:

Self reminder: 1: No Local Turfs, 2: K.I.S.S., 3: Time, 4: This is HUB tools., 5: No wasted effort.

My cut at "must have" items are marked with `*'. Items marked with `-` should be cut from FCS.)

Large scale hosting:

Synchronous
* Full client chat features PLUS
* Telepathy
* Crowds
* Amphitheater (now a standard feature of competing platforms.)
? Broadcast audio
? Observers.
- Audience participation tools (voting, volunteer assistants, questions, etc.)
Asynchhronous
* ID Secure web-based BBS.
* Web-based help.
? Answering machine (Added value. Could be moved to client side using email *)
? Download/Upload area (web-based)
- Secure email.

Operations:

* Limited scope certificate issuance (TOS compliance only.)
* Banish/Gag/Etc. tool (in-world.)
* TOS creation, modification and publication tools.

Group/Membership management:

* Full end user group management tool set (esp. address book & capabilities) PLUS
* Simple customer database
* Directory services
* Billing model control (shared billing records, overhead accounts, etc.)
* Historical transaction file/database (AKA the Bad Boys file.)

Service identity/Branding:

* Full end user tool set (includes texturing and positioning) PLUS
* Watermark creation
* Object art/sound/model creation/importation
* Geography creation/importation
? Object exportation & update (to use with outside tools)
- Region/Realm exportation & update
- Avatar model creation/importation
- Object behavior composition
- Object behavior authoring

Revenue:

? Customer activity tracking (audit tool.)
? Billing infrastructure.

Possible added value features:

* Turf hosting (hub defined TOS)
? Title company management (scarce items, ownership.)
? Internal currency management (tokens, bank, etc.)
- Turf hosting (turf defined TOS)
- Large scale customer database
- Arbitrary certificate creation & issuance

Open questions:

  • Authoring regions with a 3D tool is one-way. Shouldn't we focus on making object creation two way instead?
  • Initially, can't EC be the sole provider of scarcity and tokens for other added value services? As partners require, these services can be bundled/shipped separately.
  • Likewise, we may be able to delay allowing hubs to create new certificate types, if that is of any benefit.
  • Note that, though turfs might initially (during the no local hosting release) be forced to accept the TOS of a Hub/Microcosm, the individual hubs can't have this restriction, as they aren't hosted/connected to a higher server. TOS tools can not be excluded from the FCS hub release.
